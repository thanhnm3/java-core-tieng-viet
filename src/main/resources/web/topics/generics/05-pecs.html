PECS — Producer Extends, Consumer Super

Quy tắc chọn wildcard: Producer (cung cấp/đọc) → extends; Consumer (tiêu thụ/ghi) → super.

Ví dụ: copy

Collections.copy(List<? super T> dest, List<? extends T> src) — dest nhận (consumer) → super; src cung cấp (producer) → extends.

// Đúng — PECS
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
  for (T t : src) {
    dest.add(t);
  }
}

List<Number> dest = new ArrayList<>();
List<Integer> src = List.of(1, 2, 3);
copy(dest, src);  // OK — Integer extends Number
// SAI — dùng extends cho dest
void copyWrong(List<? extends T> dest, List<? extends T> src) {
  for (T t : src) {
    dest.add(t);  // Compile error! dest là consumer
  }
}

Producer vs Consumer

Producer (extends)

  • Tham số cung cấp data
  • Chỉ đọc, iterate
  • Ví dụ: src trong copy

Consumer (super)

  • Tham số nhận data
  • Chỉ ghi, add
  • Ví dụ: dest trong copy
