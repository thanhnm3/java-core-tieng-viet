PECS — Producer Extends, Consumer Super

Kỹ thuật

Quy tắc chọn wildcard: Producer (cung cấp/đọc) → extends; Consumer (tiêu thụ/ghi) → super. Tham số produce element (src, Stream) → extends. Tham số consume element (dest, Comparator) → super.

Ví dụ: copy

Collections.copy(List<? super T> dest, List<? extends T> src) — dest nhận (consumer) → super; src cung cấp (producer) → extends.

Comparator — Consumer (super)

void sort(List<T> list, Comparator<? super T> c) — Comparator nhận T để so sánh (consumer). Dùng super: có thể sort List<Integer> với Comparator<Number> (so sánh Number).

Comparator<Number> comp = Comparator.comparing(Number::intValue);
List<Integer> list = new ArrayList<>(List.of(3, 1, 2));
list.sort(comp);  // OK — Comparator<? super Integer>

Stream — Producer (extends)

Stream<T> produce element — khi truyền collection vào API nhận Stream, dùng extends nếu API chỉ đọc. Ví dụ: void process(Stream<? extends Number> stream).

// Đúng — PECS
public static <T> void copy(List<? super T> dest, List<? extends T> src) {
  for (T t : src) {
    dest.add(t);
  }
}

List<Number> dest = new ArrayList<>();
List<Integer> src = List.of(1, 2, 3);
copy(dest, src);  // OK — Integer extends Number
// SAI — dùng extends cho dest
void copyWrong(List<? extends T> dest, List<? extends T> src) {
  for (T t : src) {
    dest.add(t);  // Compile error! dest là consumer
  }
}

Producer vs Consumer

Producer (extends)

  • Tham số cung cấp data
  • Chỉ đọc, iterate
  • Ví dụ: src trong copy, Stream

Consumer (super)

  • Tham số nhận data
  • Chỉ ghi, add
  • Ví dụ: dest trong copy, Comparator
