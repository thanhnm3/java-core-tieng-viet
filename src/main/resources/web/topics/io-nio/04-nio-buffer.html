Buffer — java.nio

Buffer là container cho dữ liệu primitive. ByteBuffer là phổ biến nhất. Có 4 thuộc tính: capacity (kích thước), position (vị trí đọc/ghi), limit (giới hạn), mark (đánh dấu để reset).

Trạng thái Buffer — position, limit, capacity

0 ≤ mark ≤ position ≤ limit ≤ capacity. Write mode: position = vị trí ghi tiếp, limit = capacity. Read mode (sau flip): position = 0, limit = số byte đã ghi.

// Write: put() tăng position
// [pos=0, limit=cap] → put() → [pos=5, limit=cap]

// flip(): chuyển sang read
// limit=position, position=0
// [pos=0, limit=5]

// Read: get() tăng position
// clear(): position=0, limit=capacity (chuẩn bị ghi lại)

Cơ chế flip, clear, compact

Để ghi: put() tăng position. Sau khi ghi xong, gọi flip() — limit = position, position = 0. Để đọc: get() lấy từ buffer. Sau khi đọc xong, clear() — position = 0, limit = capacity (chuẩn bị ghi lại). compact() — đẩy dữ liệu chưa đọc lên đầu, position = remaining, limit = capacity.

Buffer methods
Method Thay đổi Mục đích
flip() limit=position, position=0 Chuyển từ write sang read
clear() position=0, limit=capacity Chuẩn bị ghi mới
compact() Copy remaining lên đầu Giữ dữ liệu chưa đọc, ghi tiếp
rewind() position=0 Đọc lại từ đầu

wrap, duplicate, slice

ByteBuffer.wrap(byte[]) — buffer wrap mảng, thay đổi buffer ảnh hưởng mảng. buffer.duplicate() — buffer mới chia sẻ data, position/limit độc lập. buffer.slice() — buffer con từ position đến limit, chia sẻ data.

byte[] arr = {1, 2, 3, 4, 5};
ByteBuffer buf = ByteBuffer.wrap(arr);
buf.put(0, (byte) 10);  // arr[0] = 10

ByteBuffer dup = buf.duplicate();  // Cùng data, position riêng
ByteBuffer slice = buf.slice();    // Từ position đến limit

asReadOnlyBuffer

buffer.asReadOnlyBuffer() — buffer read-only, put() ném ReadOnlyBufferException. Dùng khi truyền buffer cho code khác mà không muốn bị sửa.

ByteBuffer buf = ByteBuffer.allocate(1024);

// Ghi
buf.put("Hello".getBytes(StandardCharsets.UTF_8));
buf.flip();  // Chuyển sang mode đọc

// Đọc
byte[] dst = new byte[buf.remaining()];
buf.get(dst);
System.out.println(new String(dst, StandardCharsets.UTF_8));

// Direct buffer — nằm ngoài heap, tốt cho I/O
ByteBuffer direct = ByteBuffer.allocateDirect(1024);

Heap vs Direct Buffer

allocate() — buffer trên heap (byte[]). allocateDirect() — buffer ngoài heap, JVM có thể dùng DMA cho I/O, tránh copy qua kernel. Direct buffer tốt cho high-throughput nhưng allocation tốn hơn.

Heap vs Direct Buffer
Tiêu chí Heap Direct
Vị trí JVM heap Ngoài heap (native)
Allocation Nhanh Chậm hơn
I/O Copy qua kernel Zero-copy (DMA)
Dùng khi Mặc định, buffer nhỏ High-throughput, buffer lớn
