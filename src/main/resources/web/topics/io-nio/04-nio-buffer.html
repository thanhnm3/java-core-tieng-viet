Buffer — java.nio

Buffer là container cho dữ liệu primitive. ByteBuffer là phổ biến nhất. Có 4 thuộc tính: capacity (kích thước), position (vị trí đọc/ghi), limit (giới hạn), mark (đánh dấu để reset).

Cơ chế flip

Để ghi: put() tăng position. Sau khi ghi xong, gọi flip() — limit = position, position = 0. Để đọc: get() lấy từ buffer. Sau khi đọc xong, clear() — position = 0, limit = capacity (chuẩn bị ghi lại). compact() — đẩy dữ liệu chưa đọc lên đầu, position = remaining, limit = capacity.

Buffer methods
Method Thay đổi Mục đích
flip() limit=position, position=0 Chuyển từ write sang read
clear() position=0, limit=capacity Chuẩn bị ghi mới
compact() Copy remaining lên đầu Giữ dữ liệu chưa đọc, ghi tiếp
rewind() position=0 Đọc lại từ đầu 
ByteBuffer buf = ByteBuffer.allocate(1024);

// Ghi
buf.put("Hello".getBytes(StandardCharsets.UTF_8));
buf.flip();  // Chuyển sang mode đọc

// Đọc
byte[] dst = new byte[buf.remaining()];
buf.get(dst);
System.out.println(new String(dst, StandardCharsets.UTF_8));

// Direct buffer — nằm ngoài heap, tốt cho I/O
ByteBuffer direct = ByteBuffer.allocateDirect(1024);

Heap vs Direct Buffer

allocate() — buffer trên heap (byte[]). allocateDirect() — buffer ngoài heap, JVM có thể dùng DMA cho I/O, tránh copy qua kernel. Direct buffer tốt cho high-throughput nhưng allocation tốn hơn.

Heap vs Direct Buffer

Heap Buffer

  • Allocation nhanh
  • GC quản lý
  • Dùng mặc định

Direct Buffer

  • Zero-copy I/O với kernel
  • Phù hợp high-throughput
  • Allocation chậm
