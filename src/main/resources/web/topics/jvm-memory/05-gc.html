Garbage Collection — Cơ chế sâu

GC thu hồi Object unreachable. Hai loại: Tracing (mark từ root) và Reference Counting (Java không dùng — không xử lý cycle).

GC Roots

Điểm bắt đầu trace: (1) Local variable trong Stack frame đang thực thi; (2) Static variable; (3) JNI global reference; (4) Object được lock (monitor). Từ Root, trace qua reference → Object reachable. Còn lại = unreachable.

Mark-Sweep-Compact

Mark: DFS/BFS từ Root, đánh dấu live. Sweep: Duyệt Heap, free unreachable. Compact: Dồn live object — giảm fragmentation, tăng locality. Compact tốn thời gian — nhiều GC tối ưu để giảm compact.

G1 (Garbage First)

Chia Heap thành region (~2048 region, mỗi region 1–32MB). Mỗi region là Eden, Survivor, hoặc Old. Ưu tiên dọn region có nhiều rác (garbage first). Evacuation: copy live object từ region này sang region khác. Mixed GC: dọn cả Young và một phần Old. Pause time mục tiêu: -XX:MaxGCPauseMillis (mặc định 200ms).

ZGC

Concurrent, pause < 1ms bất kể Heap size. Dùng colored pointers: 64-bit pointer chứa metadata (mark, remap) trong bit cao — không cần separate mark bitmap. Load barrier khi đọc reference. Concurrent marking, concurrent relocation. Heap có thể TB.

Shenandoah

Concurrent evacuation — copy Object trong khi application chạy. Forwarding pointer: Object cũ trỏ tới bản copy mới. Read barrier. Pause không phụ thuộc Heap size.

G1 Mark concurrent, Evacuation STW Vài ms – 200ms Đến vài chục GB
ZGC Full concurrent < 1ms Đến TB
Shenandoah Full concurrent < 1ms Lớn
