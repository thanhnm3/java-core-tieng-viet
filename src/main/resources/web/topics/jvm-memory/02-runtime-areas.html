Runtime Data Areas — Chi tiết kỹ thuật

Kỹ thuật

Bộ nhớ JVM chia 5 khu vực: Shared (Heap, Metaspace) và Per-Thread (Stack, PC Register, Native Method Stack).

Bản đồ bộ nhớ

┌──────────────────────────────────────┐
│ Metaspace (Native Memory)            │  ← Class metadata
├──────────────────────────────────────┤
│ Heap                                 │
│   Young (Eden + S0 + S1)             │
│   Old                                 │
├──────────────────────────────────────┤
│ Stack (per thread)                    │  ← Frames
│ PC Register (per thread)             │
│ Native Method Stack (per thread)      │
└──────────────────────────────────────┘

Heap — Cấu trúc Object

Mỗi Object trên Heap có Object Header: Mark Word (8 byte trên 64-bit) — hash, GC age, lock state (biased, lightweight, heavyweight); Klass Pointer — trỏ tới metadata trong Metaspace. Sau header là instance fields (theo thứ tự declaration, alignment). Array có thêm length field (4 byte).

Ví dụ — Object layout

class Point {
  int x;   // 4 byte
  int y;   // 4 byte
}
// Trên Heap: [Mark 8B][Klass 4B][x 4B][y 4B] + alignment
// Tổng: 12 (header) + 8 (fields) = 20 → align 24 byte

Metaspace vs PermGen

Java 8 bỏ PermGen (nằm trong Heap, giới hạn -XX:MaxPermSize). Metaspace nằm trong Native Memory — không giới hạn bởi -Xmx. Lưu: class metadata (Klass), method metadata, constant pool, annotations. Class loader bị GC → metadata tương ứng có thể unload. -XX:MaxMetaspaceSize giới hạn để tránh Native OOM.

Stack — Stack Frame

Mỗi lời gọi method tạo Stack Frame: Local Variables (slot cho biến cục bộ, slot 0 = this với instance method); Operand Stack (LIFO cho bytecode thao tác); Frame Data (return address, exception table). Kích thước frame cố định tại compile time. Stack overflow khi đệ quy sâu hoặc frame quá lớn.

PC Register

Lưu địa chỉ (offset) lệnh JVM hiện tại. Native method: PC undefined. Mỗi thread một PC.
