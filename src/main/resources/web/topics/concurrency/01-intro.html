Concurrency — Lập trình đa luồng

Concurrency giúp ứng dụng tận dụng CPU nhiều nhân. Ví dụ quán cà phê: thay vì một người làm tất cả (pha chế, bưng bê, tính tiền), thuê nhiều nhân viên làm cùng lúc. Java cung cấp Thread, Executor, synchronized, và java.util.concurrent để xử lý đa luồng an toàn.

Mỗi phần gồm: Kỹ thuật (định nghĩa, cơ chế) và Ví dụ (code minh họa).

Process và Thread

Kỹ thuật

Process (tiến trình) — chương trình đang chạy, có vùng nhớ riêng. Ví dụ: một instance IntelliJ IDEA là một Process. Thread (luồng) — đơn vị nhỏ nhất trong Process. Các Thread trong cùng Process chia sẻ Heap nhưng có Stack riêng.

Process vs Thread
Tiêu chí Process Thread
Vùng nhớ Riêng biệt Chia sẻ Heap, Stack riêng
Giao tiếp IPC (phức tạp) Chia sẻ biến (cần sync)
Chi phí tạo Cao Thấp hơn

Daemon Thread

Daemon thread chạy nền — JVM thoát khi tất cả user thread kết thúc, không đợi daemon. Ví dụ: GC thread. thread.setDaemon(true) phải gọi trước start().

Thread daemon = new Thread(() -> {
  while (true) {
    cleanup();  // Chạy nền
    Thread.sleep(1000);
  }
});
daemon.setDaemon(true);  // Trước start()
daemon.start();
// JVM thoát khi main thread kết thúc, không đợi daemon

Thread Priority

Thread.MIN_PRIORITY (1) đến Thread.MAX_PRIORITY (10), mặc định 5. Chỉ gợi ý cho scheduler — không đảm bảo. Tránh phụ thuộc priority cho logic.

Ba cách tạo Thread trong Java

1. Extends class Thread

Đơn giản nhưng không tối ưu — Java không cho phép đa kế thừa, nên class không thể extend thêm class khác.

class MyThread extends Thread {
  @Override
  public void run() {
    System.out.println("Running: " + getName());
  }
}
new MyThread().start();

2. Implements Runnable

Cách phổ biến nhất. Tách biệt logic thực thi và quản lý luồng. Có thể implement thêm interface khác.

Runnable task = () -> System.out.println("Thread: " + Thread.currentThread().getName());
Thread t = new Thread(task);
t.start();

// Hoặc với tên thread
Thread t2 = new Thread(task, "Worker-1");
t2.start();

3. Implements Callable

Giống Runnable nhưng trả về kết quả (Future) và có thể ném Exception. Dùng với ExecutorService.submit().

Callable<Integer> task = () -> {
  Thread.sleep(1000);
  return 42;
};
ExecutorService exec = Executors.newSingleThreadExecutor();
Future<Integer> future = exec.submit(task);
System.out.println(future.get());  // 42 — block cho đến khi xong
exec.shutdown();
So sánh 3 cách tạo Thread
Cách Trả về Exception Khuyến nghị
extends Thread — — Tránh (đa kế thừa)
implements Runnable — Phải catch trong run() Phổ biến
implements Callable Future Ném ra ngoài Khi cần kết quả

Lưu ý

Runnable: Dùng cho task không cần kết quả; Callable khi cần Future; ưu tiên ExecutorService thay vì new Thread().

Tránh: extends Thread — mất cơ hội kế thừa class khác; tạo quá nhiều thread thủ công — dùng thread pool.
