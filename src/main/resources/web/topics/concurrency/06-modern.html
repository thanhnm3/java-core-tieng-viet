Modern Java Concurrency

CompletableFuture (Java 8)

Lập trình bất đồng bộ theo phong cách Functional. Kết chuỗi: "Làm A xong → lấy kết quả làm B → lỗi thì C". supplyAsync, thenApply, thenCompose, thenCombine, exceptionally, handle.

CompletableFuture<String> future = CompletableFuture
  .supplyAsync(() -> fetchData())
  .thenApply(s -> process(s))
  .exceptionally(ex -> "fallback");

String result = future.get();

// Kết hợp 2 future
CompletableFuture<Integer> a = CompletableFuture.supplyAsync(() -> 1);
CompletableFuture<Integer> b = CompletableFuture.supplyAsync(() -> 2);
CompletableFuture<Integer> sum = a.thenCombine(b, Integer::sum);  // 3

// Chạy song song nhiều task
CompletableFuture.allOf(f1, f2, f3).join();

Fork/Join Framework

Chia nhỏ tác vụ lớn thành tác vụ con, chạy song song. Thuật toán work-stealing: thread rỗi "đánh cắp" task từ thread bận. ForkJoinPool, RecursiveTask<T> (có return), RecursiveAction (không return). Dùng trong parallelStream().

class SumTask extends RecursiveTask<Long> {
  private final int[] arr;
  private final int from, to;

  SumTask(int[] arr, int from, int to) {
    this.arr = arr;
    this.from = from;
    this.to = to;
  }

  @Override
  protected Long compute() {
    if (to - from <= 1000) {
      long sum = 0;
      for (int i = from; i < to; i++) sum += arr[i];
      return sum;
    }
    int mid = (from + to) / 2;
    SumTask left = new SumTask(arr, from, mid);
    SumTask right = new SumTask(arr, mid, to);
    left.fork();
    return right.compute() + left.join();
  }
}

ForkJoinPool pool = new ForkJoinPool();
long result = pool.invoke(new SumTask(arr, 0, arr.length));

Virtual Threads (Java 21)

Luồng siêu nhẹ do JVM quản lý, không phải OS. Có thể chạy hàng triệu thread trên laptop. Blocking I/O không tốn OS thread — JVM unmount platform thread khi block. Thread.ofVirtual().start(task) hoặc Executors.newVirtualThreadPerTaskExecutor().

// Tạo virtual thread
Thread.ofVirtual().start(() -> System.out.println("Virtual thread"));

// Executor với virtual thread
ExecutorService exec = Executors.newVirtualThreadPerTaskExecutor();
exec.submit(() -> doIO());  // Mỗi task = 1 virtual thread
exec.shutdown();

// Tương thích với existing code — dùng như thread thường
try (var exec = Executors.newVirtualThreadPerTaskExecutor()) {
  IntStream.range(0, 10_000).forEach(i ->
    exec.submit(() -> {
      Thread.sleep(1000);  // Không block platform thread
      return i;
    })
  );
}
Platform Thread vs Virtual Thread
Tiêu chí Platform Thread Virtual Thread
Quản lý OS JVM
Stack ~1MB KB, mở rộng khi cần
Số lượng Hàng nghìn Hàng triệu
Blocking I/O Chiếm OS thread Unmount, không tốn

Khi nào dùng Virtual Threads?

Nên dùng: Ứng dụng nhiều I/O blocking; server HTTP; thay thế thread pool cho task I/O-bound.

Tránh khi: CPU-bound thuần — platform thread vẫn tốt; cần pin (synchronized với native); thread-local với số lượng lớn.
