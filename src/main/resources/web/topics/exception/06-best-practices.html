Best Practices

Đừng nuốt Exception

Không viết catch (Exception e) {} mà không log. Bạn đang "bịt mắt" khi hệ thống lỗi. Luôn log: log.error("Failed", e) hoặc rethrow.

Catch cụ thể

Thay vì catch (Exception e), catch FileNotFoundException, SQLException nếu biết rõ. Xử lý chính xác hơn.

Custom Exception

Dự án lớn: tạo Exception riêng (InsufficientBalanceException, DuplicateOrderException) để định nghĩa rõ lỗi nghiệp vụ. Kế thừa RuntimeException (unchecked) hoặc Exception (checked) tùy ngữ cảnh.

Global Exception Handler

Trong Spring Boot: @ControllerAdvice + @ExceptionHandler — tập trung xử lý lỗi, trả về response thống nhất. Business code không cần try-catch dày đặc.

// Custom Exception
public class InsufficientBalanceException extends RuntimeException {
  public InsufficientBalanceException(String message) {
    super(message);
  }
  public InsufficientBalanceException(String message, Throwable cause) {
    super(message, cause);
  }
}
