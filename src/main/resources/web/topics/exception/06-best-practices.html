Best Practices

Đừng nuốt Exception

Không viết catch (Exception e) {} mà không log. Bạn đang "bịt mắt" khi hệ thống lỗi. Luôn log: log.error("Failed", e) hoặc rethrow. log.error("Failed: " + e.getMessage()) — mất stack trace, tránh.

Catch cụ thể

Thay vì catch (Exception e), catch FileNotFoundException, SQLException nếu biết rõ. Xử lý chính xác hơn. Catch Exception chỉ ở layer cao (controller, main) để tránh crash.

Exception message — Mô tả rõ

Message nên mô tả: cái gì lỗi, ở đâu, giá trị gây lỗi. Ví dụ: "Invalid age: " + age + ", must be 0-150". Tránh message chung chung: "Error", "Failed".

throw new IllegalArgumentException(
  "Invalid userId: " + userId + ", must be positive");

Custom Exception

Dự án lớn: tạo Exception riêng (InsufficientBalanceException, DuplicateOrderException) để định nghĩa rõ lỗi nghiệp vụ. Kế thừa RuntimeException (unchecked) hoặc Exception (checked) tùy ngữ cảnh. Luôn có constructor nhận (String message, Throwable cause).

Global Exception Handler

Trong Spring Boot: @ControllerAdvice + @ExceptionHandler — tập trung xử lý lỗi, trả về response thống nhất. Business code không cần try-catch dày đặc.

@ControllerAdvice
public class GlobalExceptionHandler {
  @ExceptionHandler(IllegalArgumentException.class)
  public ResponseEntity<ErrorResponse> handleBadRequest(
      IllegalArgumentException e) {
    return ResponseEntity.badRequest()
        .body(new ErrorResponse(e.getMessage()));
  }
}
