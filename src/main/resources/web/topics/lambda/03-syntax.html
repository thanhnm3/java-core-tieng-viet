Cú pháp Lambda và Method Reference

Cú pháp Lambda

Các dạng cú pháp
Dạng Ví dụ
Không tham số () -> 42 hoặc () -> { return 42; }
1 tham số (có thể bỏ ngoặc) x -> x * 2 hoặc (x) -> x * 2
Nhiều tham số (a, b) -> a + b
Khai báo kiểu (int x, int y) -> x + y
Block body (x) -> { int r = x * 2; return r; }

Method Reference

Rút gọn lambda khi body chỉ gọi một method. Có 4 loại. Method reference không thực thi ngay — chỉ tham chiếu đến method, thực thi khi được gọi.

4 loại method reference
Loại Cú pháp Ví dụ
Static ClassName::staticMethod Integer::parseInt thay s -> Integer.parseInt(s)
Instance (object cố định) object::instanceMethod str::equals thay s -> str.equals(s)
Instance (tham số là receiver) ClassName::instanceMethod String::length thay s -> s.length()
Constructor ClassName::new ArrayList::new thay () -> new ArrayList<>()

Method reference — Overload resolution

Khi có nhiều overload, compiler chọn dựa trên target type. Ví dụ: String::valueOf có thể là IntFunction (valueOf(int)) hoặc Function<Object,String> (valueOf(Object)) — tùy context.

// Compiler chọn overload phù hợp target type
IntFunction<String> f1 = String::valueOf;           // valueOf(int)
Function<Object, String> f2 = String::valueOf;     // valueOf(Object)
// Static
List<String> nums = List.of("1", "2", "3");
nums.stream().map(Integer::parseInt).toList();  // [1, 2, 3]

// Instance (receiver = tham số)
nums.stream().map(String::toUpperCase).toList();

// Instance (object cố định)
String prefix = "id-";
nums.stream().map(prefix::concat).toList();  // ["id-1", "id-2", ...]

// Constructor
Supplier<List<String>> listSupplier = ArrayList::new;

Effectively final và capture scope

Biến dùng trong lambda phải effectively final — không gán lại sau khi được capture. Khác với final explicit: effectively final cho phép biến chưa khai báo final nhưng không thay đổi sau khi gán. Lambda capture biến theo value (bản sao tham chiếu) tại thời điểm capture.

int count = 0;
list.forEach(s -> {
  count++;  // Compile error! count không effectively final
});

// Cách đúng — dùng array hoặc AtomicInteger
int[] counter = {0};
list.forEach(s -> counter[0]++);  // Mutable reference, effectively final

// Hoặc AtomicInteger (thread-safe nếu parallel)
AtomicInteger ai = new AtomicInteger(0);
list.forEach(s -> ai.incrementAndGet());

Lambda vs Method reference

  • Method reference khi body chỉ gọi 1 method — ngắn gọn hơn
  • Lambda khi cần truyền thêm argument: n -> n * 2 thay vì method ref
  • Method reference với overload phức tạp — có thể ambiguous, dùng lambda rõ ràng hơn
