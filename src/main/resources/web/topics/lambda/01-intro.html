Lambda — Giới thiệu

Lambda expression (Java 8+) cho phép truyền hành vi như một giá trị — viết code ngắn gọn, dễ đọc thay cho anonymous class. Cú pháp: (params) -> body. Target type được suy ra từ context (assignment, method parameter, return).

Tổng quan từng phần

Mỗi phần gồm: Kỹ thuật (định nghĩa, cú pháp, quy tắc) và Ví dụ (code minh họa).

Tại sao cần Lambda?

Trước Java 8: dùng anonymous class để truyền callback — verbose, nhiều boilerplate. Lambda rút gọn logic xuống còn biểu thức. Giảm số dòng code, tăng khả năng đọc.

Kỹ thuật — Target type và SAM

Lambda không có kiểu riêng — kiểu được suy ra từ target type (biến gán, tham số method, return). Target type phải là functional interface (SAM: Single Abstract Method) — interface có đúng 1 abstract method. Ví dụ: Runnable, Comparator<T>, Predicate<T>.

// Target type từ assignment
Runnable r = () -> System.out.println("hi");  // Runnable

// Target type từ method parameter
list.removeIf(s -> s.isEmpty());  // Predicate<String>

// Target type từ return
Supplier<Integer> get = () -> 42;

Ví dụ

Anonymous class vs Lambda
Anonymous class Lambda
Nhiều dòng, boilerplate Ngắn gọn, biểu thức
new Runnable() { public void run() { ... } } () -> { ... }
Capture biến qua final Effectively final — compiler kiểm tra
Tạo class file riêng (.class) Invokedynamic — JVM tạo implementation động 
// Anonymous class — verbose
Runnable r = new Runnable() {
  @Override
  public void run() {
    System.out.println("Hello");
  }
};

// Lambda — tương đương
Runnable r = () -> System.out.println("Hello");
// Comparator — trước vs sau
List<String> names = List.of("Bob", "Alice", "Charlie");

// Anonymous
names.sort(new Comparator<String>() {
  @Override
  public int compare(String a, String b) {
    return a.length() - b.length();
  }
});

// Lambda
names.sort((a, b) -> a.length() - b.length());

Điều kiện sử dụng Lambda

  • Chỉ dùng được khi target type là functional interface — interface có đúng 1 abstract method (SAM). Default/static method không ảnh hưởng.
  • Biến được capture phải effectively final (không gán lại sau khi capture).
  • Lambda không tạo scope mới cho biến — không shadow biến ngoài.
  • Không thể dùng lambda khi target type ambiguous — ví dụ: overloaded method cần cast rõ ràng.
// Ambiguous — cần cast
// foo(x -> x)  // Nếu có foo(Predicate) và foo(Function)
foo((Predicate<Integer>) x -> x > 0);

Khi nào dùng Lambda?

  • Callback, event handler (Runnable, Consumer)
  • Filter, map, sort trong Collections/Stream (Predicate, Function, Comparator)
  • Lazy evaluation (Supplier)
  • Logic phức tạp nhiều dòng — tách method riêng
  • Cần debug chi tiết — anonymous class có tên rõ hơn trong stack trace
