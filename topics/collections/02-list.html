List — Danh sách có thứ tự

Kỹ thuật

List cho phép phần tử trùng lặp, có thứ tự (index). Hai implementation phổ biến: ArrayList (mảng động) và LinkedList (danh sách liên kết đôi).

ArrayList — Cấu trúc nội bộ

ArrayList lưu trong mảng Object[] elementData. Khi mảng đầy, tạo mảng mới lớn hơn (capacity mới = oldCapacity + (oldCapacity >> 1) — tăng ~50%), copy phần tử sang. Capacity mặc định ban đầu 10 (từ Java 8). Truy cập get(i) là elementData[i] — O(1). Thêm ở cuối add(e) O(1) amortized; thêm ở index i phải dịch n-i phần tử — O(n).

ensureCapacity và trimToSize

ensureCapacity(minCapacity) tăng capacity trước khi add nhiều phần tử — tránh resize nhiều lần. trimToSize() thu nhỏ mảng về đúng size — tiết kiệm bộ nhớ khi không còn add thêm.

Ví dụ

ArrayList<String> list = new ArrayList<>();
list.ensureCapacity(1000);  // Tránh resize khi add 1000 phần tử
for (int i = 0; i < 1000; i++) {
  list.add("item" + i);
}
list.trimToSize();  // Thu nhỏ về đúng 1000

LinkedList — Cấu trúc nội bộ

LinkedList dùng node: mỗi node có item, prev, next. Có first và last trỏ đầu/cuối. Thêm/xóa đầu hoặc cuối chỉ cần cập nhật vài reference — O(1). Truy cập index i phải duyệt từ first (hoặc last nếu i gần cuối) — O(n). Mỗi node tốn thêm ~24 byte (object header + 3 reference) so với ArrayList chỉ lưu reference trong mảng.

Độ phức tạp thời gian — ArrayList vs LinkedList
Thao tác ArrayList LinkedList
get(i) O(1) O(n)
add(e) — thêm cuối O(1) amortized O(1)
add(i, e) — chèn tại index O(n) O(n) — tìm vị trí + O(1) chèn
addFirst(e) / addLast(e) O(n) — phải dịch toàn bộ O(1)
remove(i) O(n) O(n)
removeFirst() / removeLast() O(n) O(1)
contains(o) O(n) O(n)

subList — View con của List

list.subList(fromIndex, toIndex) trả về view (không copy) — thay đổi subList ảnh hưởng list gốc và ngược lại. Dùng để xử lý một đoạn mà không tạo bản sao.

List<Integer> list = new ArrayList<>(List.of(1, 2, 3, 4, 5));
List<Integer> sub = list.subList(1, 4);  // [2, 3, 4]
sub.set(0, 20);  // list thành [1, 20, 3, 4, 5]
list.remove(2);  // Structural change — subList invalid, sub dùng lại sẽ CME

List.of() và Arrays.asList()

List.of("a", "b") — immutable, không null. Arrays.asList(arr) — view của mảng, fixed-size (không add/remove nhưng set được). Thay đổi arr ảnh hưởng list.

Ví dụ sử dụng

List<String> list = new ArrayList<>();
list.add("apple");
list.add("banana");
list.add("cherry");
list.add(1, "berry");  // Chèn tại index 1 — O(n)

System.out.println(list.get(2));  // cherry — O(1)
list.remove(0);  // Xóa "apple" — O(n)
list.set(0, "blueberry");  // Thay thế — O(1)
List<Integer> linked = new LinkedList<>();
linked.addFirst(10);
linked.addLast(20);
linked.addFirst(5);  // O(1)

System.out.println(linked.getFirst());  // 5 — O(1)
linked.removeLast();  // O(1)
linked.get(1);  // O(n) — phải duyệt từ đầu

ArrayList vs LinkedList — Bảng so sánh

Tiêu chí ArrayList LinkedList
Cấu trúc Mảng động (Object[]) Danh sách liên kết đôi
Bộ nhớ Ít hơn (chỉ mảng + reference) Nhiều hơn (mỗi node ~40 byte)
Cache locality Tốt Kém
Implement List List, Deque

Khi nào chọn List implementation?

ArrayList: Hầu hết trường hợp — get/set theo index nhanh; add cuối thường xuyên; ít tốn bộ nhớ; cache-friendly.

LinkedList: Tránh khi cần get(i) thường xuyên; ưu tiên khi cần queue/stack (addFirst/addLast/removeFirst/removeLast) hoặc chèn/xóa ở đầu/cuối rất nhiều.
