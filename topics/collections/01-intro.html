Collections - Bộ sưu tập

Java Collections Framework (JCF) nằm trong java.util, cung cấp các cấu trúc dữ liệu tái sử dụng để lưu trữ và xử lý nhóm đối tượng. Framework tách interface (contract) khỏi implementation (cách lưu trữ) — lập trình theo interface giúp dễ thay đổi implementation mà không đổi code gọi.

Tổng quan hierarchy

Collection là interface gốc cho List, Set, Queue. Map không kế thừa Collection vì cấu trúc key-value khác. Các interface con mở rộng contract (ví dụ List thêm index-based access).

Interface Thứ tự Trùng lặp Implementation chính
List Có (index) Cho phép ArrayList, LinkedList
Set Không (HashSet) / Chèn (LinkedHashSet) / Sắp xếp (TreeSet) Không HashSet, LinkedHashSet, TreeSet
Queue FIFO hoặc Priority Cho phép LinkedList, ArrayDeque, PriorityQueue
Map Key không trùng Value có thể trùng HashMap, LinkedHashMap, TreeMap

Iterator — Duyệt collection

Iterator<E> là design pattern cho phép duyệt collection mà không cần biết cấu trúc nội bộ. Interface có 3 method: hasNext(), next(), remove(). Iterable<T> có method iterator() — for-each compile thành gọi iterator().

Cơ chế hoạt động

Iterator giữ con trỏ (cursor) tới vị trí hiện tại. next() trả về phần tử và tiến cursor. remove() xóa phần tử vừa trả về bởi next() — gọi remove() hai lần liên tiếp không có next() ở giữa sẽ ném IllegalStateException.

Fail-fast và ConcurrentModificationException

Iterator của ArrayList, HashMap... là fail-fast: nếu collection bị sửa (add/remove) ngoài iterator trong lúc duyệt, sẽ ném ConcurrentModificationException. Cách an toàn: dùng iterator.remove() thay vì list.remove() trong vòng lặp.

List<String> list = new ArrayList<>(List.of("a", "b", "c"));

// For-each (dùng Iterator bên trong)
for (String s : list) {
  System.out.println(s);
}

// Iterator tường minh — remove an toàn
Iterator<String> it = list.iterator();
while (it.hasNext()) {
  String s = it.next();
  if (s.equals("b")) {
    it.remove();  // OK — không gây ConcurrentModificationException
  }
}

// SAI: list.remove(s) trong for-each → ConcurrentModificationException

Khi nào dùng Iterator?

Nên dùng: Cần remove trong lúc duyệt; duyệt Set (không có index); tránh ConcurrentModificationException; ListIterator khi cần duyệt ngược hoặc set phần tử.

Tránh khi: Chỉ duyệt đơn giản — for-each ngắn gọn hơn; List cần truy cập index — dùng for(i) với get(i).
