Queue và Deque

Queue — FIFO; Deque — thêm/xóa cả hai đầu. Implementation: LinkedList, ArrayDeque, PriorityQueue.

ArrayDeque — Mảng vòng (circular array)

Lưu trong Object[] elements. Hai chỉ số head và tail. Index thực tế: (head + i) & (elements.length - 1) — dùng bit AND vì length luôn lũy thừa 2. offerLast: ghi tail, tăng tail; offerFirst: giảm head, ghi head. Khi đầy: capacity x2, copy. Không cho phép null. Nhanh hơn LinkedList vì: không tạo node, cache locality tốt, ít allocation.

ArrayDeque — Queue và Stack

ArrayDeque implement cả Queue và Deque. Dùng offer(e)/poll() cho FIFO queue. Dùng push(e)/pop() (tương đương addFirst/removeFirst) cho LIFO stack. Tránh class Stack (legacy) — ArrayDeque nhanh hơn và thread-safe hơn khi dùng đúng cách.

// Queue (FIFO)
Queue<String> queue = new ArrayDeque<>();
queue.offer("first");
queue.offer("second");
queue.offer("third");
System.out.println(queue.poll());  // first
System.out.println(queue.poll());  // second

// Stack (LIFO)
Deque<String> stack = new ArrayDeque<>();
stack.push("a");
stack.push("b");
stack.push("c");
System.out.println(stack.pop());  // c
System.out.println(stack.peek()); // b (không xóa)

PriorityQueue — Binary Heap

Lưu trong mảng Object[] queue biểu diễn min-heap (hoặc max-heap với Comparator.reverseOrder()). Parent tại (i-1)/2, left child 2*i+1, right child 2*i+2. offer: thêm cuối, siftUp (heapify up); poll: lấy root, đặt phần tử cuối lên root, siftDown (heapify down). Cả hai O(log n). peek O(1). Iterator không đảm bảo thứ tự — chỉ poll mới lấy theo thứ tự.

PriorityQueue — Custom Comparator

// Min-heap (mặc định)
Queue<Integer> minHeap = new PriorityQueue<>();

// Max-heap
Queue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());

// Custom: sắp xếp theo độ ưu tiên
record Task(String name, int priority) { }
Queue<Task> tasks = new PriorityQueue<>(
  Comparator.comparingInt(Task::priority).reversed()
);
tasks.offer(new Task("low", 1));
tasks.offer(new Task("high", 10));
System.out.println(tasks.poll().name());  // high

BlockingQueue — Đa luồng

BlockingQueue (trong java.util.concurrent) — put/take chặn khi queue đầy/rỗng. ArrayBlockingQueue, LinkedBlockingQueue, PriorityBlockingQueue. Dùng cho producer-consumer pattern.

BlockingQueue<String> queue = new LinkedBlockingQueue<>(10);

// Producer
queue.put("item");  // Chặn nếu đầy

// Consumer
String item = queue.take();  // Chặn nếu rỗng

// Non-blocking
queue.offer("item");  // false nếu đầy
String s = queue.poll();  // null nếu rỗng
Độ phức tạp — LinkedList vs ArrayDeque vs PriorityQueue
Thao tác LinkedList ArrayDeque PriorityQueue
offer(e) / poll() O(1) O(1) amortized O(log n)
peek() O(1) O(1) O(1)
offerFirst(e) / offerLast(e) O(1) O(1) amortized —
pollFirst() / pollLast() O(1) O(1) amortized —
Cấu trúc Doubly-linked list Circular array Binary heap (array)
Thứ tự lấy FIFO FIFO Priority (min/max) 
Deque<Integer> deque = new ArrayDeque<>();
deque.offerFirst(1);
deque.offerLast(2);
deque.offerFirst(0);
System.out.println(deque.pollFirst());  // 0
System.out.println(deque.pollLast());   // 2
Queue<Integer> pq = new PriorityQueue<>();
pq.offer(30);
pq.offer(10);
pq.offer(20);
System.out.println(pq.poll());  // 10 (min-heap)
System.out.println(pq.poll());  // 20

// Max-heap
Queue<String> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
maxHeap.offer("a");
maxHeap.offer("c");
maxHeap.offer("b");
System.out.println(maxHeap.poll());  // c

Queue vs Stack — Khi nào dùng

Cấu trúc Thứ tự Use case Implementation
Queue FIFO BFS, task queue, producer-consumer ArrayDeque, LinkedList
Stack LIFO DFS, undo, biểu thức, parse ArrayDeque (tránh class Stack cũ)
PriorityQueue Priority Dijkstra, A*, merge k sorted lists PriorityQueue

Khi nào chọn Queue implementation?

ArrayDeque: Queue hoặc stack thông thường — nhanh, ít bộ nhớ; PriorityQueue khi cần lấy theo ưu tiên (heap).

LinkedList: Tránh khi chỉ dùng queue/stack — ArrayDeque tốt hơn; class Stack (legacy) — dùng ArrayDeque thay thế.

Collections — Tóm tắt

Ưu điểm: Không cần implement từ đầu; type-safe với generics; tối ưu, đã test kỹ; chọn implementation theo độ phức tạp và use case.
Lưu ý: Hiểu đặc điểm từng loại (O(1) vs O(log n)); custom object phải equals/hashCode đúng; immutable key cho Map khi có thể.
