Trang chủ / Map

Map — Key-Value

Map lưu cặp key-value; key không trùng (equals). Ba implementation: HashMap (hash table), LinkedHashMap (hash + thứ tự), TreeMap (Red-Black tree).

HashMap — Cấu trúc nội bộ

HashMap dùng mảng Node[] table. Mỗi bucket là Node (key, value, next) — linked list. Index: (n - 1) & hash(key) với hash = (h = key.hashCode()) ^ (h >>> 16) để trộn bit cao xuống thấp. Capacity luôn lũy thừa 2 (mặc định 16).

Xử lý collision

Chaining: Nhiều entry cùng bucket nối thành linked list. Treeify (Java 8+): Khi bucket có > 8 node và table.length >= 64, chuyển sang TreeNode (Red-Black tree) — worst case O(log n) thay vì O(n). Untreeify khi bucket còn < 6 node.

Load factor và resize

Load factor = số phần tử / capacity. Mặc định 0.75. Khi size > threshold (= capacity * loadFactor) thì resize: capacity x2, rehash toàn bộ. Resize tốn O(n) nhưng amortized vẫn O(1) cho put.

Độ phức tạp — HashMap vs LinkedHashMap vs TreeMap
Thao tác HashMap LinkedHashMap TreeMap
get(key) O(1) avg O(1) avg O(log n)
put(key, value) O(1) avg O(1) avg O(log n)
remove(key) O(1) avg O(1) avg O(log n)
firstKey() / lastKey() O(n) O(n) O(log n)
subMap(from, to) — — O(log n) để lấy view
Null key Cho phép (1 cái) Cho phép Không (natural order)

LinkedHashMap — Access order

Constructor LinkedHashMap(capacity, loadFactor, true) — tham số thứ 3 = accessOrder. Khi true: mỗi lần get/put, entry được chuyển về cuối linked list. Dùng cho LRU cache: entry ít dùng ở đầu, có thể removeEldestEntry khi vượt capacity.

Map<String, Integer> map = new HashMap<>();
map.put("apple", 1);
map.put("banana", 2);
map.put("banana", 20);  // Ghi đè

// LRU cache với max 3 phần tử
Map<String, String> lru = new LinkedHashMap<>(3, 0.75f, true) {
  @Override
  protected boolean removeEldestEntry(Map.Entry<String, String> e) {
    return size() > 3;
  }
};

Khi nào chọn Map implementation?

HashMap: Mặc định — O(1) lookup; LinkedHashMap khi cần thứ tự chèn hoặc LRU; TreeMap khi cần range query (subMap, firstKey, lastKey) hoặc iteration theo thứ tự key.

TreeMap: Tránh khi không cần thứ tự — HashMap nhanh hơn; key phải Comparable hoặc truyền Comparator; null key gây NPE với natural order.
