Set — Tập hợp không trùng

Set không cho phép phần tử trùng lặp — xác định bằng equals(). Ba implementation: HashSet (hash table), LinkedHashSet (hash + linked list giữ thứ tự), TreeSet (Red-Black tree).

HashSet — Cấu trúc nội bộ

HashSet bên trong dùng HashMap — key là phần tử Set, value là dummy object PRESENT. add/contains/remove gọi put/get/remove của HashMap. Bucket index = hash(key) & (n-1) (n = capacity, lũy thừa 2). Collision xử lý bằng chaining (linked list) hoặc tree (Java 8+: khi bucket > 8 phần tử chuyển sang TreeNode). Load factor mặc định 0.75 — khi vượt quá sẽ resize (capacity x2) và rehash.

equals và hashCode — Contract bắt buộc

Với custom object trong Set/Map: equals so sánh logic; hashCode phải nhất quán với equals — nếu a.equals(b) thì a.hashCode() == b.hashCode(). Ngược lại không đúng (hash collision). Nếu hashCode sai: hai object equals nhưng hash khác nhau → có thể cả hai vào Set (vi phạm không trùng). Nếu hashCode đúng nhưng equals sai: hai object không equals nhưng hash trùng → chaining, vẫn đúng.

Ví dụ override equals và hashCode

public class Person {
  private final String id;
  private final String name;

  public Person(String id, String name) {
    this.id = id;
    this.name = name;
  }

  @Override
  public boolean equals(Object o) {
    if (this == o) return true;
    if (!(o instanceof Person person)) return false;
    return id.equals(person.id);
  }

  @Override
  public int hashCode() {
    return id.hashCode();  // Dùng cùng field với equals
  }
}

// Hoặc dùng record (Java 16+) — tự sinh equals/hashCode
record PersonRecord(String id, String name) { }

Set<Person> set = new HashSet<>();
set.add(new Person("1", "Alice"));
set.add(new Person("1", "Alice"));  // Bị bỏ qua (trùng id)
set.add(new Person("2", "Bob"));
System.out.println(set.size());  // 2

LinkedHashSet

Kế thừa HashSet, thêm doubly-linked list chạy qua các entry theo thứ tự chèn. Mỗi entry có before, after. Iteration theo thứ tự thêm vào. Overhead nhỏ so với HashSet.

TreeSet

Dùng TreeMap (Red-Black tree) bên trong. Phần tử phải Comparable hoặc truyền Comparator. Cây cân bằng — độ cao O(log n). first(), last(), headSet(), tailSet() — O(log n). Không cho phép null khi dùng natural order.

EnumSet — Set cho enum (hiệu năng cao)

EnumSet dùng bit vector — rất nhanh, ít bộ nhớ. Chỉ dùng với enum. EnumSet.of(), EnumSet.allOf(), EnumSet.range().

enum Day { MON, TUE, WED, THU, FRI, SAT, SUN }

Set<Day> weekend = EnumSet.of(Day.SAT, Day.SUN);
Set<Day> weekdays = EnumSet.range(Day.MON, Day.FRI);
Set<Day> all = EnumSet.allOf(Day.class);

Set operations — Union, intersection, difference

Set<Integer> a = new HashSet<>(Set.of(1, 2, 3));
Set<Integer> b = new HashSet<>(Set.of(2, 3, 4));

// Union
Set<Integer> union = new HashSet<>(a);
union.addAll(b);  // {1, 2, 3, 4}

// Intersection
Set<Integer> inter = new HashSet<>(a);
inter.retainAll(b);  // {2, 3}

// Difference (a - b)
Set<Integer> diff = new HashSet<>(a);
diff.removeAll(b);  // {1}
Độ phức tạp — HashSet vs LinkedHashSet vs TreeSet
Thao tác HashSet LinkedHashSet TreeSet
add(e) O(1) avg O(1) avg O(log n)
contains(e) O(1) avg O(1) avg O(log n)
remove(e) O(1) avg O(1) avg O(log n)
first() / last() O(n) O(n) O(log n)
Thứ tự iteration Không đảm bảo Insertion order Sorted (Comparable/Comparator) 
Set<String> set = new HashSet<>();
set.add("apple");
set.add("banana");
set.add("apple");  // Bị bỏ qua
System.out.println(set.size());  // 2

Set<Integer> tree = new TreeSet<>(Comparator.reverseOrder());
tree.addAll(List.of(30, 10, 20));  // 30, 20, 10
System.out.println(tree.first());  // 30

Set — Ưu và nhược điểm

Ưu điểm: contains O(1) hoặc O(log n); đảm bảo không trùng; loại duplicate; kiểm tra tồn tại nhanh.
Nhược điểm: Không có index; custom object phải implement equals/hashCode đúng; TreeSet không cho null (natural order).
