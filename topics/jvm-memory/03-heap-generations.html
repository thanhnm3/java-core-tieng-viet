Trang chủ / Heap Generations

Generational Heap — Chi tiết

Dựa trên Weak Generational Hypothesis: phần lớn Object chết trẻ. Young Gen nhỏ, Minor GC nhanh; Old Gen lớn, Major GC chậm nhưng ít xảy ra.

TLAB (Thread-Local Allocation Buffer)

Mỗi thread có vùng nhỏ trong Eden để allocate Object — tránh lock khi nhiều thread cùng new. Object lớn (vượt TLAB) allocate trực tiếp Eden. -XX:+UseTLAB mặc định bật.

Young Generation — Eden, Survivor

Eden: Mọi allocation mới. Khi đầy → Minor GC. Object sống copy sang Survivor (Copying algorithm). S0, S1: Chỉ một active; mỗi Minor GC: copy survivor từ S0→S1 hoặc ngược lại, clear vùng cũ. Object sống qua N lần Minor GC (tenuring threshold, mặc định 15) → promote lên Old. -XX:MaxTenuringThreshold.

Old Generation

Object promoted từ Young. Major/Full GC dọn ở đây — thường Stop-the-World lâu hơn. G1 chia Heap thành region (1–32MB), không còn Eden/S0/S1 cố định — mỗi region có thể Eden, Survivor, hoặc Old.

Thuật toán GC theo vùng
Vùng Thuật toán Đặc điểm
Young (Eden, Survivor) Copying Copy live object, clear toàn bộ — không fragmentation
Old Mark-Sweep-Compact Mark live, sweep dead, compact để giảm fragmentation

Xây dựng bằng Java 17+ · Không dùng framework · Pure NIO
