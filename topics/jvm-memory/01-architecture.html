JVM & Memory — Đầu não của Java

Hiểu JVM và quản lý bộ nhớ giúp giải quyết OutOfMemoryError, StackOverflowError và tối ưu hiệu năng. JVM chia làm 3 phân vùng chính.

Kiến trúc tổng quan JVM

Kỹ thuật — Sơ đồ tổng quan

┌─────────────────────────────────────────────────────────┐
│ Class Loader Subsystem (Load → Link → Initialize)       │
├─────────────────────────────────────────────────────────┤
│ Runtime Data Areas                                       │
│   Heap (shared) │ Metaspace │ Stack (per-thread) │ PC   │
├─────────────────────────────────────────────────────────┤
│ Execution Engine (Interpreter + JIT + GC)                │
└─────────────────────────────────────────────────────────┘

1. Class Loader Subsystem

Load: Đọc file .class thành byte array, tạo Class object trong Heap. Link gồm 3 bước: Verify (bytecode hợp lệ, không vi phạm type), Prepare (cấp phát bộ nhớ cho static, gán default), Resolve (chuyển symbolic reference thành direct reference — có thể lazy). Initialize: Thực thi <clinit> (static block, static field init).

Parent Delegation Model

Class loader con ủy quyền cha load trước. Bootstrap (null) → Extension → Application. Đảm bảo class từ rt.jar luôn do Bootstrap load — tránh override core class. findClass() chỉ gọi khi cha không tìm thấy.

Ví dụ

// Thứ tự: Application → Extension → Bootstrap
// Bootstrap load: java.lang.*, java.util.*...
// Application load: class của ứng dụng

// Custom ClassLoader override findClass(), không loadClass()
// loadClass() gọi parent.loadClass() trước

2. Runtime Data Areas

Heap, Metaspace, Stack, PC Register, Native Method Stack. Chi tiết ở section sau.

3. Execution Engine

Interpreter: Thông dịch bytecode từng lệnh — nhanh khởi động, chậm khi chạy lâu. JIT (Just-In-Time): Compile bytecode → native code khi method "nóng" (hot). GC: Thu hồi Object unreachable.

JIT Compilation — Tiered

C1 (Client): Compile nhanh, ít optimization — dùng cho method ít gọi. C2 (Server): Compile chậm, tối ưu mạnh (inlining, loop unrolling, escape analysis). Tiered: C1 trước (level 1–3), C2 cho hot code (level 4). Threshold: -XX:CompileThreshold (C1: 1500, C2: 10000 invocations).

// -XX:+TieredCompilation (mặc định)
// Level 0: Interpreter
// Level 1–3: C1 compiled
// Level 4: C2 compiled (optimized)
