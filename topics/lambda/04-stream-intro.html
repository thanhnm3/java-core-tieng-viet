Stream API — Giới thiệu

Stream cho phép xử lý collection theo kiểu declarative — mô tả cái gì cần làm, không phải cách làm. Hỗ trợ pipeline, lazy evaluation, và có thể chạy song song.

Tạo Stream

// Từ collection
List<Integer> list = List.of(1, 2, 3);
Stream<Integer> s1 = list.stream();
Stream<Integer> s2 = list.parallelStream();

// Từ array
int[] arr = {1, 2, 3};
IntStream is = Arrays.stream(arr);

// Từ giá trị
Stream<String> s3 = Stream.of("a", "b", "c");

// Vô hạn
Stream<Integer> infinite = Stream.iterate(0, n -> n + 1);
Stream<Double> random = Stream.generate(Math::random);

Intermediate vs Terminal

Phân loại operation
Intermediate Terminal
Trả về Stream, có thể chain Trả về kết quả hoặc void
Lazy — không thực thi cho đến khi có terminal Eager — trigger toàn bộ pipeline
filter, map, flatMap, sorted, distinct, limit, skip forEach, collect, reduce, count, findFirst, anyMatch

Lazy Evaluation

Intermediate operations không chạy ngay — chúng chỉ build pipeline. Khi gọi terminal, pipeline mới được thực thi từ đầu đến cuối.

List<Integer> src = List.of(1, 2, 3, 4, 5);
src.stream()
   .filter(n -> {
     System.out.println("filter:" + n);  // Chỉ in khi terminal chạy
     return n % 2 == 0;
   })
   .map(n -> {
     System.out.println("map:" + n);
     return n * 2;
   })
   .findFirst();  // Terminal — trigger pipeline
// Output: filter:1, filter:2, map:2 — dừng ngay khi có kết quả

Stream vs for-loop

Stream

  • Declarative, dễ đọc
  • Lazy, có thể short-circuit
  • Dễ parallel (parallelStream)

for-loop

  • Imperative, kiểm soát chi tiết
  • Overhead nhỏ hơn cho collection nhỏ
  • Phù hợp khi cần break/continue phức tạp
