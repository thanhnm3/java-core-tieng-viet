4. Polymorphism (Đa hình)

Định nghĩa

Đa hình cho phép biến kiểu cha tham chiếu tới object kiểu con. Khi gọi phương thức, JVM sẽ gọi implementation của object thực tế (runtime polymorphism). Java có hai loại: override (runtime — cùng signature, khác implementation) và overload (compile-time — cùng tên, khác tham số).

Runtime Polymorphism — Override

Biến kiểu cha trỏ tới object kiểu con. Gọi method: JVM xác định implementation dựa trên kiểu thực tế của object (runtime type), không phải kiểu biến (compile-time type).

Person p = new Student("Bob", 20, "S001");
p.introduce();  // Gọi Student.introduce() — đa hình runtime

// p có kiểu compile-time: Person
// p trỏ tới object kiểu runtime: Student
// JVM gọi method của Student

Override vs Overload

Tiêu chí Override Overload
Quan hệ Cha-con (inheritance) Cùng class
Signature Giống hệt (tên + tham số) Khác tham số (số lượng hoặc kiểu)
Thời điểm chọn Runtime (dynamic binding) Compile-time (static binding)

Overload — Nạp chồng phương thức

Overload: nhiều method cùng tên, khác danh sách tham số. Compiler chọn method dựa trên kiểu và số lượng tham số tại compile-time.

public class Calculator {
  public int add(int a, int b) {
    return a + b;
  }

  public double add(double a, double b) {
    return a + b;
  }

  public int add(int a, int b, int c) {
    return a + b + c;
  }
}

Calculator calc = new Calculator();
calc.add(1, 2);       // Gọi add(int, int) → 3
calc.add(1.0, 2.0);   // Gọi add(double, double) → 3.0
calc.add(1, 2, 3);   // Gọi add(int, int, int) → 6

Ví dụ đa hình trong thực tế

public void processShapes(List<Shape> shapes) {
  for (Shape s : shapes) {
    System.out.println("Area: " + s.area());
    // s có thể là Circle, Rectangle, Triangle...
    // Mỗi loại gọi area() của chính nó
  }
}

List<Shape> list = List.of(
  new Circle(5),
  new Rectangle(3, 4)
);
processShapes(list);  // Không cần if-else theo từng loại

Ép kiểu (Casting) khi cần method riêng

Khi biến kiểu cha cần gọi method chỉ có ở class con, phải cast. Dùng instanceof kiểm tra trước để tránh ClassCastException.

Person p = new Student("Alice", 20, "S001");

if (p instanceof Student s) {  // Pattern matching (Java 16+)
  System.out.println(s.getStudentId());
}

// Hoặc cast tường minh
if (p instanceof Student) {
  Student s = (Student) p;
  System.out.println(s.getStudentId());
}

Ưu và nhược điểm của đa hình

Ưu điểm: Code linh hoạt, dễ mở rộng; dependency inversion (phụ thuộc vào abstraction); giảm điều kiện if-else.
Nhược điểm: Cần hiểu hierarchy; debug phức tạp hơn khi có nhiều implementation.

Gọi qua kiểu cha vs kiểu con

Gọi qua kiểu cha (đa hình)

Dùng khi xử lý nhiều loại object cùng interface; code không phụ thuộc implementation cụ thể.

void printName(Person p) {
  System.out.println(p.getName());
}
printName(new Student(...));
printName(new Person(...));

Gọi cụ thể kiểu con

Dùng khi cần method riêng của subclass; nhưng code bị gắn chặt với implementation.

Student s = new Student(...);
s.getStudentId(); // Chỉ Student có

Xây dựng bằng Java 17+ · Không dùng framework · Pure NIO
