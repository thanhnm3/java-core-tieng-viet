2. Encapsulation (Đóng gói)

Định nghĩa

Đóng gói là ẩn dữ liệu bên trong class, chỉ cho phép truy cập qua các phương thức công khai (getter/setter). Dùng private cho thuộc tính, public cho phương thức cần thiết. Điều này giúp kiểm soát cách dữ liệu được đọc và ghi, có thể thêm validation trong setter.

Access modifiers — Phạm vi truy cập

Modifier Class Package Subclass Mọi nơi
private Có Không Không Không
package-private (mặc định) Có Có Không Không
protected Có Có Có Không
public Có Có Có Có

Getter và Setter — Kiểm soát truy cập

Getter trả về giá trị; setter gán giá trị và có thể validate. Quy ước đặt tên: getXxx(), setXxx(value). Với boolean: isXxx(), hasXxx().

public class BankAccount {
  private double balance;
  private String owner;

  public BankAccount(String owner, double initialBalance) {
    this.owner = owner;
    if (initialBalance >= 0) {
      this.balance = initialBalance;
    }
  }

  // Getter — read-only, có thể tính toán
  public double getBalance() {
    return balance;
  }

  // Setter — validation trước khi gán
  public void setBalance(double amount) {
    if (amount >= 0) {
      this.balance = amount;
    }
    // Bỏ qua nếu amount < 0 — hoặc ném IllegalArgumentException
  }

  public void deposit(double amount) {
    if (amount > 0) {
      this.balance += amount;
    }
  }

  public boolean withdraw(double amount) {
    if (amount > 0 && amount <= balance) {
      balance -= amount;
      return true;
    }
    return false;
  }
}

Validation trong Setter

Setter là nơi đặt validation để đảm bảo dữ liệu luôn hợp lệ. Có thể ném IllegalArgumentException khi giá trị không hợp lệ.

public class Person {
  private String name;
  private int age;

  public void setAge(int age) {
    if (age < 0 || age > 150) {
      throw new IllegalArgumentException("Age must be 0-150: " + age);
    }
    this.age = age;
  }

  public void setName(String name) {
    if (name == null || name.isBlank()) {
      throw new IllegalArgumentException("Name cannot be null or blank");
    }
    this.name = name.trim();
  }
}

Read-only và Write-only

Chỉ getter: field read-only (immutable từ bên ngoài). Chỉ setter: hiếm dùng, thường cho config. Cả hai: full control.

// Read-only — chỉ có getter, không setter
public class ImmutablePoint {
  private final int x;
  private final int y;

  public ImmutablePoint(int x, int y) {
    this.x = x;
    this.y = y;
  }

  public int getX() { return x; }
  public int getY() { return y; }
  // Không có setX/setY — không thể thay đổi sau khi tạo
}

Encapsulation: Nên dùng hay không?

Khi dùng đóng gói (private + getter/setter)

  • Kiểm soát gán giá trị (validation trong setter)
  • Dễ đổi implementation nội bộ mà không ảnh hưởng code bên ngoài
  • API rõ ràng, client chỉ gọi method thay vì truy cập trực tiếp field

Khi không đóng gói (public field)

  • Có thể gán person.age = -100 từ bất kỳ đâu
  • Khó bảo trì khi logic thay đổi (phải sửa nhiều chỗ)
  • Vi phạm nguyên tắc thiết kế, dễ phát sinh bug
// Anti-pattern: tránh làm vậy
person.age = -1;
