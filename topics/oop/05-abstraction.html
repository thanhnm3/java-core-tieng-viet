5. Abstraction (Trừu tượng hóa)

Định nghĩa

Trừu tượng hóa là ẩn chi tiết triển khai, chỉ expose hành vi cần thiết qua interface hoặc abstract class. Trong Java: abstract class có thể chứa method trừu tượng và code chung; interface (từ Java 8) định nghĩa contract, có thể có default/static method.

Abstract Class

Class khai báo abstract không thể new trực tiếp. Có thể có: abstract method (không có body, subclass phải implement); concrete method (có body, dùng chung); constructor; fields.

public abstract class Animal {
  protected String name;

  public Animal(String name) {
    this.name = name;
  }

  // Abstract method — subclass BẮT BUỘC implement
  public abstract void makeSound();

  // Concrete method — dùng chung
  public void sleep() {
    System.out.println(name + " is sleeping... Zzz");
  }
}

public class Dog extends Animal {
  public Dog(String name) {
    super(name);
  }

  @Override
  public void makeSound() {
    System.out.println(name + " says: Woof!");
  }
}

Interface

Interface định nghĩa contract — class implement phải cung cấp tất cả method (trừ default/static). Từ Java 8: default method (có body, optional override); static method. Từ Java 9: private method. Một class có thể implement nhiều interface.

public interface Drawable {
  void draw();  // Abstract — implement bắt buộc

  default void highlight() {
    System.out.println("Highlighted");
  }

  static void printInfo() {
    System.out.println("Drawable interface");
  }
}

public class Circle implements Drawable {
  @Override
  public void draw() {
    System.out.println("Drawing circle");
  }
  // highlight() không cần override — dùng default
}

Drawable.printInfo();  // Gọi static qua interface

Abstract Class vs Interface

Tiêu chí Abstract Class Interface
Kế thừa Một class (extends) Nhiều interface (implements)
Constructor Có Không
Fields Có (kể cả non-static) Chỉ public static final (constant)
Method body Có (concrete method) default, static, private (từ Java 8/9)
Dùng khi Chia sẻ code, quan hệ is-a Contract, nhiều hành vi độc lập

Ví dụ kết hợp Interface + Abstract Class

public interface Flyable {
  void fly();
}

public abstract class Bird implements Flyable {
  protected String name;

  public Bird(String name) {
    this.name = name;
  }

  public abstract void makeSound();

  @Override
  public void fly() {
    System.out.println(name + " is flying");
  }
}

public class Sparrow extends Bird {
  public Sparrow(String name) {
    super(name);
  }

  @Override
  public void makeSound() {
    System.out.println("Chirp!");
  }
}

Khi nào dùng abstract class vs interface?

Abstract class: Cần chia sẻ code chung (fields, methods) cho các subclass; quan hệ "is-a" với logic dùng chung.

Interface: Định nghĩa contract mà nhiều class không liên quan có thể implement; Java không đa kế thừa class nên dùng interface để "implement" nhiều hành vi.

Ưu và nhược điểm của abstraction

Ưu điểm: Giảm coupling; dễ thay implementation; code phụ thuộc abstraction thay vì chi tiết cụ thể.
Nhược điểm: Thêm lớp indirection; có thể over-abstract nếu hierarchy quá phức tạp.

Tóm tắt: Bốn trụ cột OOP bổ trợ nhau. Encapsulation bảo vệ dữ liệu; Inheritance tái sử dụng và mở rộng; Polymorphism cho phép xử lý linh hoạt; Abstraction ẩn chi tiết triển khai. Dùng đúng chỗ, tránh lạm dụng — đặc biệt kế thừa khi composition phù hợp hơn.
