Exception Propagation

Nếu Exception xảy ra trong method mà không được catch, nó được đẩy ngược lên (pop) khỏi Stack Frame của method đó và chuyển về method gọi nó. Quá trình tiếp tục cho đến khi gặp catch phù hợp hoặc lên main(). Nếu không ai xử lý — JVM in stack trace và dừng Thread đó.

Stack Unwinding

JVM "unwind" Stack: pop các Frame cho đến khi tìm thấy handler. Mỗi Frame bị pop — local variable mất, nhưng Exception object vẫn giữ trong Heap. Stack trace ghi lại chuỗi Frame (call stack) tại thời điểm throw.

Stack Trace

e.printStackTrace() — in ra stderr. e.getStackTrace() — mảng StackTraceElement. e.getCause() — Exception gốc (khi wrap). Log đầy đủ: log.error("msg", e) — logger in cả stack trace.

try {
  doWork();
} catch (Exception e) {
  log.error("Failed", e);  // In cả stack trace
  // e.printStackTrace();  // Tránh — dùng logger
}

Chained Exception — initCause

Constructor Exception(String, Throwable) — wrap cause. Hoặc initCause(Throwable) khi Exception không có constructor nhận cause. getCause() lấy cause. Giữ nguyên nguyên nhân khi rethrow — giúp debug.

try {
  parse(input);
} catch (NumberFormatException e) {
  ServiceException ex = new ServiceException("Parse failed");
  ex.initCause(e);  // Khi không có constructor (cause)
  throw ex;
}

// Hoặc dùng constructor
throw new IOException("wrap", cause);
void a() { b(); }
void b() { c(); }
void c() {
  throw new RuntimeException("error");
}
// Propagation: c → b → a → main → JVM in stack trace

Rethrow

Nên: Catch, log, rethrow với cause: throw new ServiceException("failed", e) — giữ stack trace gốc.
Tránh: throw new Exception(e.getMessage()) — mất cause, mất stack trace gốc.
