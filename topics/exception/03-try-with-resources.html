Try-with-resources (Java 7+)

Tự động đóng resource implement AutoCloseable (có method close()). Tránh memory leak khi quên close trong finally. Resource được đóng theo thứ tự ngược với khai báo.

Cơ chế

Compiler sinh bytecode: gọi close() trong block finally ẩn. Nếu cả try và close() đều ném Exception — Exception từ try được suppressed, có thể lấy qua getSuppressed(). Java 9+: biến trong try-with-resources có thể là effectively final.

// Java 7+
try (BufferedReader br = new BufferedReader(
    new FileReader("test.txt"))) {
  System.out.println(br.readLine());
} catch (IOException e) {
  e.printStackTrace();
}
// br.close() tự động gọi, dù có lỗi hay không
// Nhiều resource — đóng theo thứ tự ngược
try (InputStream in = openInput();
     OutputStream out = openOutput()) {
  copy(in, out);
}
// out.close() trước, rồi in.close()

Khi nào dùng

Luôn dùng: InputStream, OutputStream, Reader, Writer, Connection, Socket — mọi resource cần close.

Tránh: Gọi close() thủ công trong finally khi đã có try-with-resources — dư thừa, dễ lỗi.
