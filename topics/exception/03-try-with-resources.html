Try-with-resources (Java 7+)

Tự động đóng resource implement AutoCloseable (có method close()). Tránh memory leak khi quên close trong finally. Resource được đóng theo thứ tự ngược với khai báo.

AutoCloseable vs Closeable

AutoCloseable (Java 7) — close() throws Exception. Closeable extends AutoCloseable — close() throws IOException. InputStream, Reader implement Closeable. Try-with-resources chấp nhận cả hai.

Cơ chế

Compiler sinh bytecode: gọi close() trong block finally ẩn. Nếu cả try và close() đều ném Exception — Exception từ try được suppressed, có thể lấy qua getSuppressed(). Java 9+: biến trong try-with-resources có thể là effectively final (không cần khai báo trong ngoặc).

Suppressed Exceptions

Khi try block ném Exception và close() cũng ném — Exception gốc được ném ra, Exception từ close() được add vào getSuppressed(). Log đầy đủ: e.getSuppressed().

try (InputStream in = new FileInputStream("x")) {
  in.read();  // Ném IOException
} catch (IOException e) {
  // e: IOException từ read()
  for (Throwable t : e.getSuppressed()) {
    // t: IOException từ close()
  }
}

Java 9+ — Effectively final

Biến có thể khai báo ngoài try, miễn là effectively final.

// Java 9+
InputStream in = openStream();
OutputStream out = openStream();
try (in; out) {
  copy(in, out);
}
// Java 7+
try (BufferedReader br = new BufferedReader(
    new FileReader("test.txt"))) {
  System.out.println(br.readLine());
} catch (IOException e) {
  e.printStackTrace();
}
// br.close() tự động gọi, dù có lỗi hay không
// Nhiều resource — đóng theo thứ tự ngược
try (InputStream in = openInput();
     OutputStream out = openOutput()) {
  copy(in, out);
}
// out.close() trước, rồi in.close()

Khi nào dùng

Luôn dùng: InputStream, OutputStream, Reader, Writer, Connection, Socket — mọi resource cần close.

Tránh: Gọi close() thủ công trong finally khi đã có try-with-resources — dư thừa, dễ lỗi.
