Type Erasure và Giới hạn

Kỹ thuật

Generics chỉ tồn tại tại compile-time. JVM không biết generic — type erasure: compiler xóa thông tin type, thay bằng bound (hoặc Object nếu unbounded). List<String> và List<Integer> cùng class tại runtime. Bridge method: compiler sinh method để tương thích override. Reifiable: type còn tại runtime (raw, ?). Non-reifiable: List<String> — mất thông tin.

Erasure — Sau compile

Ví dụ — Erasure

// Trước
class Box<T> { T value; }
class NumBox<T extends Number> { T value; }

// Sau erasure
class Box { Object value; }      // T → Object
class NumBox { Number value; }    // T → Number

Bridge method

Khi override method generic, compiler sinh bridge method để tương thích binary. Ví dụ: class MyList implements List<String> — override add(String) nhưng interface có add(Object) — bridge gọi add(String).

Không new T[]

new T[10] — lỗi (T không tồn tại runtime). Giải pháp: (1) (T[]) new Object[n] — unchecked cast, có thể ClassCastException nếu array thoát ra ngoài; (2) Array.newInstance(Class<T> clazz, int n) — an toàn; (3) dùng List<T> thay vì array.

Ví dụ — new T[]

// Cách 2 — an toàn
public static <T> T[] createArray(Class<T> clazz, int size) {
  @SuppressWarnings("unchecked")
  T[] arr = (T[]) java.lang.reflect.Array.newInstance(clazz, size);
  return arr;
}
String[] arr = createArray(String.class, 10);

Reifiable vs Non-reifiable

Reifiable: Type còn thông tin tại runtime — raw type, unbounded wildcard, primitive, array. Non-reifiable: List<String>, List<? extends Number> — erasure mất thông tin. Không dùng non-reifiable trong instanceof, cast (trừ unbounded), new.

Hậu quả của Erasure

  • Không instanceof: if (obj instanceof List<String>) — lỗi. Chỉ obj instanceof List.
  • Không new T[]: new T[10] — lỗi. Dùng Array.newInstance hoặc List.
  • Không primitive: List<int> — lỗi. Dùng List<Integer>.
  • Overload theo T: void m(List<String>) và void m(List<Integer>) — cùng erasure, compile error.
  • Static field T: static T instance — lỗi. Mỗi instantiation chia sẻ cùng static.
Giới hạn Generics
Giới hạn Giải pháp
Không primitive Integer, Long... (boxing)
Không new T[] Array.newInstance(Class, n) hoặc List
Không instanceof T instanceof List (raw)
Không static field T Mỗi instantiation chia sẻ cùng static

Generics — Tóm tắt

Ưu điểm: Type-safe, không cast, lỗi compile-time; code tái sử dụng; Collections API.
Giới hạn: Erasure — mất type tại runtime; không primitive; cú pháp phức tạp với wildcard.
