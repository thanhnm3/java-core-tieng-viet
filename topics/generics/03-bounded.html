Bounded Type Parameters

Kỹ thuật

Giới hạn type parameter: <T extends Number> — T phải là Number hoặc subclass. Cho phép gọi method của Number: t.intValue(). extends dùng cho cả class và interface (Java không có implements trong generic bound). Multiple bounds: <T extends A & B> — class đứng trước, tối đa 1 class. Recursive bound: <T extends Comparable<T>> — T so sánh được với chính nó.

extends cho interface

Trong bound, extends áp dụng cho cả interface: <T extends Comparable<T>>. T phải implement Comparable. Nhiều interface: <T extends Serializable & Cloneable>.

Multiple bounds

<T extends A & B & C> — T phải thỏa tất cả. Class đứng trước (nếu có), tối đa 1 class: <T extends Number & Comparable<T>>. Thứ tự sai: <T extends Comparable & Number> — compile error.

Recursive type bound

<T extends Comparable<T>> — T phải so sánh được với chính nó. Dùng cho max, min: T max(T a, T b) cần a.compareTo(b).

Ví dụ

public static <T extends Comparable<T>> T max(T a, T b) {
  return a.compareTo(b) > 0 ? a : b;
}
max(10, 20);      // 20
max("a", "b");    // "b"
// Bounded — chỉ chấp nhận Number
public static <T extends Number> double sum(List<T> list) {
  double total = 0;
  for (T n : list) {
    total += n.doubleValue();  // Có method của Number
  }
  return total;
}
sum(List.of(1, 2, 3));     // OK
sum(List.of(1.5, 2.5));    // OK
sum(List.of("a"));         // Compile error
// Multiple bounds — class trước
<T extends Number & Comparable<T>> T max(T a, T b) {
  return a.compareTo(b) > 0 ? a : b;
}

Khi nào dùng bound?

Nên dùng: Cần gọi method trên T; giới hạn kiểu chấp nhận (Number, Comparable).

Tránh: Bound quá chặt khi không cần — hạn chế flexibility.
