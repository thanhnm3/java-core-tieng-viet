Generic Class và Method

Kỹ thuật — Generic Class

Khai báo type parameter trong class: class Box<T> { private T value; }. T là type variable — thay thế bằng kiểu cụ thể khi dùng: Box<String>, Box<Integer>. Có thể có nhiều parameter: class Pair<K, V>. Quy ước đặt tên: T (type), K (key), V (value), E (element), R (return).

Nhiều type parameter

Map<K, V>, BiFunction<T, U, R> — mỗi parameter độc lập. Dùng chữ in hoa: T, K, V, E (element), R (return).

class Pair<K, V> {
  private K key;
  private V value;
  public Pair(K k, V v) { key = k; value = v; }
  public K getKey() { return key; }
  public V getValue() { return value; }
}
Pair<String, Integer> p = new Pair<>("age", 30);

Kỹ thuật — Generic Method

Type parameter khai báo trước return type: <T> T getFirst(List<T> list). Compiler infer từ argument: getFirst(stringList) → T = String. Có thể dùng với class không generic.

Type inference

Compiler suy ra T từ context. List.of("a", "b") → List<String>. Collections.emptyList() → cần hint: Collections.<String>emptyList() khi context không rõ. Diamond new ArrayList<>() — infer từ biến bên trái.

// Inference từ argument
String s = getFirst(List.of("a", "b"));  // T = String

// Inference thất bại — cần explicit
List<String> empty = Collections.<String>emptyList();

Ví dụ — Generic Class

// Generic class
class Box<T> {
  private T value;
  public void set(T v) { value = v; }
  public T get() { return value; }
}
Box<Integer> box = new Box<>();
box.set(42);
int n = box.get();  // Không cast

Ví dụ — Generic Method

// Generic method
public static <T> T getFirst(List<T> list) {
  return list.isEmpty() ? null : list.get(0);
}
String s = getFirst(List.of("a", "b"));  // T inferred = String
