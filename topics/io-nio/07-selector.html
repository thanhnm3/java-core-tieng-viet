Trang chủ / Selector

Selector — Multiplexing

Selector cho phép 1 thread quản lý nhiều SelectableChannel (SocketChannel, ServerSocketChannel...). Channel phải configureBlocking(false). Đăng ký channel với selector: channel.register(selector, SelectionKey.OP_READ).

Cơ chế

selector.select() block cho đến khi có channel có sự kiện (OP_ACCEPT, OP_READ, OP_WRITE, OP_CONNECT). Trả về số key đã sẵn sàng. selector.selectedKeys() lấy set các key. Mỗi key có channel(), readyOps(). Xử lý xong phải keyIterator.remove().

SelectionKey operations
Constant Ý nghĩa
OP_ACCEPT ServerSocketChannel có kết nối mới
OP_CONNECT Kết nối thành công
OP_READ Có dữ liệu đọc
OP_WRITE Có thể ghi 
Selector selector = Selector.open();
ServerSocketChannel server = ServerSocketChannel.open();
server.configureBlocking(false);
server.bind(new InetSocketAddress(port));
server.register(selector, SelectionKey.OP_ACCEPT);

while (true) {
  selector.select();
  Set<SelectionKey> keys = selector.selectedKeys();
  Iterator<SelectionKey> it = keys.iterator();
  while (it.hasNext()) {
    SelectionKey key = it.next();
    it.remove();
    if (key.isAcceptable()) handleAccept(key);
    else if (key.isReadable()) handleRead(key);
  }
}

Blocking vs Non-blocking

Blocking (thread per connection)

  • 1 thread = 1 kết nối
  • Code đơn giản
  • Hàng nghìn kết nối = hàng nghìn thread

Non-blocking + Selector

  • 1 thread quản lý nhiều channel
  • Phù hợp server nhiều kết nối
  • Code phức tạp hơn

Khi nào dùng Selector?

Nên dùng: Server socket (HTTP, TCP) nhiều kết nối đồng thời; cần giảm số thread; high concurrency.

Tránh khi: Client đơn giản; ít kết nối; có thể dùng NIO.2 hoặc thư viện (Netty, Java 21 Virtual Thread).
