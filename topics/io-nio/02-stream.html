InputStream và OutputStream

Đọc/ghi byte. read() trả về byte (0–255) hoặc -1 khi hết; read(byte[] b) đọc vào mảng. write(int b) / write(byte[] b) ghi. Luôn dùng try-with-resources.

Cấu trúc

InputStream là abstract: read() abstract. read(byte[] b) mặc định gọi read() từng byte — chậm. Override trong BufferedInputStream để đọc block. available() ước lượng byte có sẵn (không đáng tin với network).

InputStream vs Reader
Class Đơn vị Encoding Use case
InputStream Byte Không xử lý Binary, image, file
Reader Char (16-bit) Có (InputStreamReader) Text, UTF-8 
// Đọc byte
try (InputStream in = new FileInputStream("data.bin")) {
  byte[] buf = new byte[8192];
  int n;
  while ((n = in.read(buf)) != -1) {
    process(buf, 0, n);
  }
}

// Đọc text với encoding
try (Reader r = new InputStreamReader(
    new FileInputStream("data.txt"), StandardCharsets.UTF_8)) {
  char[] buf = new char[1024];
  int n;
  while ((n = r.read(buf)) != -1) {
    System.out.print(new String(buf, 0, n));
  }
}

Stream — Ưu và nhược điểm

Ưu điểm: Đơn giản; Reader/Writer xử lý encoding; dễ wrap (BufferedInputStream, GZIPInputStream).
Nhược điểm: Blocking; mỗi kết nối cần 1 thread; không có multiplexing.

Xây dựng bằng Java 17+ · Không dùng framework · Pure NIO
