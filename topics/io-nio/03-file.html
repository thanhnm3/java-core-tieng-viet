File và Buffered Stream

File (java.io) — đại diện path, không mở file. FileInputStream / FileOutputStream — stream byte. FileReader / FileWriter — stream char (mặc định encoding hệ thống).

BufferedInputStream / BufferedOutputStream

Lớp wrapper: giữ buffer nội bộ (mặc định 8KB). read() lấy từ buffer; khi buffer rỗng mới gọi in.read() xuống đĩa. Giảm số lần system call — đọc 1KB từng byte = 1024 lần; với buffer = 1 lần.

Độ phức tạp — Unbuffered vs Buffered
Thao tác FileInputStream BufferedInputStream
read() 1 byte 1 system call Đọc block, lấy từ buffer
read(byte[]) 8KB 1 system call 1 system call (buffer size)
Đọc 1MB Nhiều call ~128 call (8KB buffer) 
// Đọc file text — luôn dùng BufferedReader
try (BufferedReader br = new BufferedReader(
    new InputStreamReader(new FileInputStream("data.txt"), StandardCharsets.UTF_8))) {
  String line;
  while ((line = br.readLine()) != null) {
    System.out.println(line);
  }
}

// Ghi file
try (BufferedWriter bw = new BufferedWriter(
    new OutputStreamWriter(new FileOutputStream("out.txt"), StandardCharsets.UTF_8))) {
  bw.write("Hello");
  bw.newLine();
}

Khi nào dùng Buffered?

Luôn dùng: BufferedInputStream/BufferedOutputStream khi đọc/ghi file; BufferedReader cho text (readLine).

Tránh: FileInputStream/FileReader trực tiếp khi đọc nhiều — chậm do không có buffer.
