Vòng đời Thread (Thread Lifecycle)

Thread không chỉ "chạy" hay "dừng" — nó đi qua các trạng thái. Mỗi trạng thái có Thread.State tương ứng.

Chuyển đổi trạng thái

NEW → start() → RUNNABLE
RUNNABLE → synchronized block bị chiếm → BLOCKED
RUNNABLE → wait() / join() → WAITING
RUNNABLE → sleep(ms) / wait(ms) / join(ms) → TIMED_WAITING
RUNNABLE/WAITING/TIMED_WAITING/BLOCKED → run() kết thúc → TERMINATED
Các trạng thái Thread
Trạng thái Mô tả
NEW Vừa tạo bằng new Thread(), chưa gọi start()
RUNNABLE Đã gọi start(), đang chờ CPU hoặc đang chạy
BLOCKED Đang chờ lấy monitor Lock (ví dụ chờ vào synchronized)
WAITING Đợi vô thời hạn cho đến khi thread khác đánh thức (wait(), join())
TIMED_WAITING Đợi có thời gian (sleep(ms), wait(ms), join(ms))
TERMINATED Đã hoàn thành hoặc bị lỗi

join() — Đợi thread kết thúc

thread.join() — thread gọi join() sẽ block cho đến khi thread đích kết thúc. join(timeout) — đợi tối đa timeout ms. Dùng khi cần đợi kết quả từ thread con.

Thread worker = new Thread(() -> doWork());
worker.start();
worker.join();  // Main thread đợi worker xong
System.out.println("Worker done");

interrupt() — Ngắt thread

thread.interrupt() đặt cờ interrupt. Thread đang sleep/wait/join sẽ nhận InterruptedException. Thread đang chạy phải kiểm tra Thread.interrupted() hoặc isInterrupted(). Luôn restore interrupt: Thread.currentThread().interrupt() sau khi catch.

Thread t = new Thread(() -> {
  try {
    while (!Thread.interrupted()) {
      doWork();
      Thread.sleep(100);
    }
  } catch (InterruptedException e) {
    Thread.currentThread().interrupt();  // Restore interrupt status
  }
});
t.start();
t.interrupt();  // Gửi tín hiệu ngắt

isAlive()

thread.isAlive() — true nếu thread đã start và chưa chết. Dùng để kiểm tra trước khi join.

Thread t = new Thread(() -> {
  try {
    Thread.sleep(1000);  // TIMED_WAITING
  } catch (InterruptedException e) {
    Thread.currentThread().interrupt();
  }
});
System.out.println(t.getState());  // NEW
t.start();
System.out.println(t.getState());  // RUNNABLE hoặc TIMED_WAITING
t.join();
System.out.println(t.getState());  // TERMINATED

Lưu ý

start() vs run(): Gọi start() để tạo luồng mới; gọi run() trực tiếp chạy trên luồng hiện tại — không tạo luồng mới.

Tránh: Gọi start() hai lần — ném IllegalThreadStateException.
