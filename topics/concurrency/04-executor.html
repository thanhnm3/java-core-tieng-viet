Executor Framework

Đừng tạo thread thủ công bằng new Thread().start() — tạo/hủy thread tốn tài nguyên. ExecutorService quản lý pool thread có sẵn, tái sử dụng.

Các loại Thread Pool

Các loại Thread Pool
Pool Đặc điểm Use case
newFixedThreadPool(n) n thread cố định, queue unbounded Workload ổn định, giới hạn tài nguyên
newCachedThreadPool() Tạo thread mới khi cần, tái sử dụng idle, 60s timeout Nhiều task ngắn, burst
newScheduledThreadPool(n) Chạy định kỳ hoặc sau delay Cron, heartbeat
newSingleThreadExecutor() 1 thread, đảm bảo thứ tự Queue tuần tự

ThreadPoolExecutor — Tùy chỉnh

Executors.newFixedThreadPool(n) thực chất tạo ThreadPoolExecutor. Có thể tạo trực tiếp với corePoolSize, maxPoolSize, keepAliveTime, queue. Khi queue đầy và chưa đạt maxPoolSize → tạo thread mới; đạt max → RejectedExecutionHandler.

ThreadPoolExecutor exec = new ThreadPoolExecutor(
  4, 10, 60, TimeUnit.SECONDS,
  new ArrayBlockingQueue<>(100),
  new ThreadPoolExecutor.CallerRunsPolicy()  // Caller chạy khi reject
);

shutdown vs shutdownNow

shutdown() — không nhận task mới, chờ task đang chạy và trong queue xong. shutdownNow() — trả về list task chưa chạy, cố dừng task đang chạy (interrupt). awaitTermination(timeout) — block cho đến khi shutdown hoặc timeout.

executor.shutdown();
if (!executor.awaitTermination(60, TimeUnit.SECONDS)) {
  executor.shutdownNow();
}

invokeAll và invokeAny

invokeAll(tasks) — chạy tất cả, trả về List<Future>, block cho đến khi xong. invokeAny(tasks) — trả về kết quả của task hoàn thành đầu tiên, hủy các task còn lại.

List<Callable<String>> tasks = List.of(
  () -> fetchFromA(),
  () -> fetchFromB()
);
List<Future<String>> futures = executor.invokeAll(tasks);

String first = executor.invokeAny(tasks);  // Lấy kết quả đầu tiên

Future.get với timeout

future.get(timeout, unit) — block tối đa timeout, ném TimeoutException nếu quá. Tránh deadlock khi task bị kẹt.

try {
  String result = future.get(5, TimeUnit.SECONDS);
} catch (TimeoutException e) {
  future.cancel(true);  // Hủy task
}
ExecutorService fixed = Executors.newFixedThreadPool(4);
fixed.submit(() -> System.out.println("Task"));

ScheduledExecutorService scheduled = Executors.newScheduledThreadPool(2);
scheduled.scheduleAtFixedRate(() -> log(), 0, 1, TimeUnit.SECONDS);

fixed.shutdown();  // Luôn shutdown khi xong

Khi nào chọn Thread Pool?

FixedThreadPool

  • Giới hạn số thread
  • Phù hợp: server xử lý request

CachedThreadPool

  • Không giới hạn (cẩn thận)
  • Phù hợp: task ngắn, burst

Lưu ý

Luôn dùng: ExecutorService thay vì new Thread(); gọi shutdown() hoặc shutdownNow() khi không dùng nữa.

Tránh: CachedThreadPool với task dài — có thể tạo quá nhiều thread.
