Executor Framework

Đừng tạo thread thủ công bằng new Thread().start() — tạo/hủy thread tốn tài nguyên. ExecutorService quản lý pool thread có sẵn, tái sử dụng.

Các loại Thread Pool
Pool Đặc điểm Use case
FixedThreadPool Số thread cố định. Workload ổn định, giới hạn tài nguyên
CachedThreadPool Tự tạo thread mới khi cần, tái sử dụng thread idle. Nhiều task ngắn, burst
ScheduledThreadPool Chạy định kỳ hoặc sau delay. Cron, heartbeat
SingleThreadExecutor Chỉ 1 thread, đảm bảo thứ tự. Queue tuần tự 
ExecutorService fixed = Executors.newFixedThreadPool(4);
fixed.submit(() -> System.out.println("Task"));

ScheduledExecutorService scheduled = Executors.newScheduledThreadPool(2);
scheduled.scheduleAtFixedRate(() -> log(), 0, 1, TimeUnit.SECONDS);

fixed.shutdown();  // Luôn shutdown khi xong

Khi nào chọn Thread Pool?

FixedThreadPool

  • Giới hạn số thread
  • Phù hợp: server xử lý request

CachedThreadPool

  • Không giới hạn (cẩn thận)
  • Phù hợp: task ngắn, burst

Lưu ý

Luôn dùng: ExecutorService thay vì new Thread(); gọi shutdown() hoặc shutdownNow() khi không dùng nữa.

Tránh: CachedThreadPool với task dài — có thể tạo quá nhiều thread.
