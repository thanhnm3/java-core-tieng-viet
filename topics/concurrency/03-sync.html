Trang chủ / Synchronized

Đồng bộ hóa (Synchronization)

Để tránh Race Condition — nhiều thread cùng sửa dữ liệu dẫn đến kết quả sai. Cần kiểm soát truy cập vào shared state.

synchronized

Khóa một phương thức hoặc block — chỉ 1 thread truy cập tại một thời điểm. Mỗi object có một monitor lock. synchronized trên method = lock trên this; trên static method = lock trên Class object.

volatile

Đảm bảo mọi thread đọc giá trị mới nhất từ main memory (không đọc từ CPU cache). Không đảm bảo atomicity cho compound (i++) — chỉ phù hợp flag đơn giản.

ReentrantLock

Linh hoạt hơn synchronized: tryLock() thử lấy lock không block; lockInterruptibly() cho phép ngắt khi đợi; lock timeout.

wait() / notify()

Giao tiếp giữa các thread. Phải gọi trong synchronized block. wait() giải phóng lock và chờ; notify() đánh thức thread đang chờ. Dùng cho Producer-Consumer.

synchronized vs volatile vs ReentrantLock
Công cụ Phạm vi tryLock Use case
synchronized Block/method Không Đơn giản, phổ biến
volatile Biến — Flag visibility
ReentrantLock Block Có Linh hoạt, timeout 
// synchronized
synchronized (lock) {
  sharedCounter++;
}

// wait/notify — Producer-Consumer
synchronized (queue) {
  while (queue.isEmpty()) {
    queue.wait();
  }
  return queue.poll();
}
// Producer: queue.offer(x); queue.notify();
