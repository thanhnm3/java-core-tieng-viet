Đồng bộ hóa (Synchronization)

Để tránh Race Condition — nhiều thread cùng sửa dữ liệu dẫn đến kết quả sai. Cần kiểm soát truy cập vào shared state.

synchronized

Khóa một phương thức hoặc block — chỉ 1 thread truy cập tại một thời điểm. Mỗi object có một monitor lock. synchronized trên method = lock trên this; trên static method = lock trên Class object. Reentrant — thread đã giữ lock có thể vào lại (đếm số lần vào).

volatile

Đảm bảo mọi thread đọc giá trị mới nhất từ main memory (không đọc từ CPU cache). Không đảm bảo atomicity cho compound (i++) — chỉ phù hợp flag đơn giản.

private volatile boolean running = true;  // Visibility

// SAI với volatile: i++ không atomic
private volatile int count;  // count++ vẫn race
// Đúng: AtomicInteger

AtomicInteger, AtomicLong, AtomicReference

Thao tác atomic không cần synchronized. incrementAndGet(), compareAndSet(), accumulateAndGet(). Dùng cho counter, flag.

AtomicInteger counter = new AtomicInteger(0);
counter.incrementAndGet();  // Atomic
counter.addAndGet(10);
counter.compareAndSet(10, 20);  // Chỉ set nếu hiện tại = 10

AtomicReference<String> ref = new AtomicReference<>("a");
ref.compareAndSet("a", "b");  // Atomic swap

ReentrantLock

Linh hoạt hơn synchronized: tryLock() thử lấy lock không block; lockInterruptibly() cho phép ngắt khi đợi; tryLock(timeout) — timeout. Luôn unlock trong finally.

ReentrantLock lock = new ReentrantLock();
if (lock.tryLock(1, TimeUnit.SECONDS)) {
  try {
    doWork();
  } finally {
    lock.unlock();
  }
}

ReentrantReadWriteLock

Nhiều thread đọc đồng thời; chỉ 1 thread ghi. Read lock không exclusive với read; write lock exclusive với mọi lock. Phù hợp: nhiều đọc, ít ghi.

ReadWriteLock rw = new ReentrantReadWriteLock();
rw.readLock().lock();
try {
  return data;  // Nhiều reader đồng thời
} finally {
  rw.readLock().unlock();
}
// Writer
rw.writeLock().lock();
try {
  data = newValue;
} finally {
  rw.writeLock().unlock();
}

CountDownLatch

Một hoặc nhiều thread đợi cho đến khi count về 0. countDown() giảm count; await() block cho đến khi 0. Dùng khi đợi N task hoàn thành.

CountDownLatch latch = new CountDownLatch(3);
for (int i = 0; i < 3; i++) {
  executor.submit(() -> {
    doWork();
    latch.countDown();
  });
}
latch.await();  // Đợi 3 task xong
System.out.println("All done");

wait() / notify()

Giao tiếp giữa các thread. Phải gọi trong synchronized block. wait() giải phóng lock và chờ; notify() đánh thức 1 thread; notifyAll() đánh thức tất cả. Dùng while thay vì if khi check điều kiện (spurious wakeup).

synchronized vs volatile vs ReentrantLock
Công cụ Phạm vi tryLock Use case
synchronized Block/method Không Đơn giản, phổ biến
volatile Biến — Flag visibility
ReentrantLock Block Có Linh hoạt, timeout
Atomic* Biến — Counter, CAS 
// synchronized
synchronized (lock) {
  sharedCounter++;
}

// wait/notify — Producer-Consumer
synchronized (queue) {
  while (queue.isEmpty()) {
    queue.wait();
  }
  return queue.poll();
}
// Producer: queue.offer(x); queue.notify();
